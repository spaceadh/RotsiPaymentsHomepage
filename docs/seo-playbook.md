# Rotsi SEO Playbook

This playbook is for the public site surfaces that matter most right now:
- homepage
- work index
- project case studies
- journal

The goal is simple: make it easier for the right people to find Rotsi through search, understand what Rotsi does quickly, and move into the right next step.

## Core Search Positioning

Rotsi should be understood as a company that builds:
- business automation systems
- custom platforms
- workflow infrastructure

The site should avoid broad, abstract positioning that searchers do not use. Search visibility improves when pages are tied to clear business problems and clear system types.

## Surface-by-Surface Focus

### Homepage

Primary job:
- establish what Rotsi builds
- connect services to real business use cases
- pass authority to work, journal, and contact

Primary language:
- business automation systems
- custom platforms
- workflow infrastructure
- websites
- WhatsApp automations
- internal platforms
- operational systems

Metadata direction:
- title should lead with the main service cluster
- description should explain what Rotsi builds and for whom
- homepage copy should mention actual workflow examples where natural

### Work Index

Primary job:
- rank for service-led and case-study-led searches
- help visitors understand the range of systems Rotsi builds

Primary language:
- case studies
- B2B commerce platform
- internal platform
- workflow automation
- technical partner
- platform design
- system architecture

Metadata direction:
- use a title that combines case studies with platform and systems language
- description should mention platforms, internal tools, automation, and real operating needs

### Project Pages

Primary job:
- rank for problem-led and system-led queries
- show how Rotsi thinks and executes
- support commercial trust

Page rules:
- each case study should target a different search intent cluster
- titles and descriptions should mention the project type and business problem
- headings should explain the challenge, mechanisms, and outcome in plain language
- proof points should be visible on the page, not hidden in metadata only

Suggested focus by project:
- Bewama: B2B ecommerce, construction materials ecommerce, abandoned cart recovery, ecommerce SEO
- Kumbusha: employee milestone reminders, WhatsApp reminder workflows, employee appreciation workflows
- Somafix Kenya Website: distributor website, product SEO website, brand visibility website
- Nexus: internal platform, customer records system, inventory visibility, role-based permissions

### Journal

Primary job:
- capture informational search demand tied to Rotsi services
- build topical authority that supports work and contact pages

Journal rules:
- every post should answer a search-shaped question
- titles should be specific and practical
- intros should clarify the task or problem quickly
- article bodies should stay close to delivery reality
- internal links should point back to relevant case studies or services where natural

## Metadata Rules

- Keep titles specific and human-readable. Do not force keywords.
- Keep descriptions clear and grounded in the actual page content.
- Use one main intent per page.
- Every important page should have:
  - title
  - description
  - canonical
  - Open Graph title and description
- Use structured data only when it matches the visible page:
  - organization schema at the site level
  - article schema on journal posts
  - page-level metadata on project pages

## Internal Linking Rules

- Homepage should link clearly to:
  - work
  - journal
  - contact
- Work index should link to every project page
- Project pages should link back to work and into contact
- Journal posts should link to relevant project pages when the topic overlaps
- Journal index should continue to act as the hub for all articles

## Content Standards

- Write for people who are trying to solve an operational problem
- Prefer concrete nouns over abstract positioning
- Show the mechanism, not just the promise
- Use proof points where approved
- Keep client-sensitive details at a safe level

## Measurement

Track progress through:
- indexed page coverage
- impressions and clicks by surface
- query coverage for project pages and journal posts
- engagement from work and journal pages into contact

The strongest sign of progress is not only traffic. It is better query match on the right pages and clearer movement from search entry into the project and contact surfaces.
