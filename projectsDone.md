This file is not public copy. It exists to preserve raw context, approved proof points, and safe positioning.

## Positioning Rule

Rotsi projects should be presented as full case studies with:
- problem
- mechanisms
- outcome

Overall narrative:
- Rotsi builds custom systems around real business workflows
- work spans UI design, software design, architecture, automation, and growth infrastructure
- each solution is tailored to a specific commercial use case rather than offered as a generic package

## Project 1: Bewama

Public name:
- Bewama

URL:
- https://bewama.com

Type:
- B2B commerce platform for construction, industrial, chemical, and related materials

Audience:
- hardware businesses
- construction buyers
- contractors
- fundis / mafundi
- DIY buyers and home repair customers

Problem:
- Needed more than a standard e-commerce storefront
- Needed a system that could attract search traffic, guide buyers through the funnel, recover intent, and support repeat purchasing
- Needed technical leadership across product design, system design, and platform execution

Rotsi role:
- UI design
- software design
- system architecture
- platform execution
- SEO implementation
- automation support for social acquisition

Mechanisms implemented:
- abandoned cart recovery
- loyalty points system tied to key engagement actions
- actions include registration, invitations/referrals, purchases, reviews, user-generated media, and daily logins
- technical SEO for products and blog content
- social automations for Facebook and Instagram including comment-to-DM flows that direct users toward the website

Approved public proof points:
- 100+ products added to the platform
- 10+ blogs published
- SEO health score can be phrased as 90%+

Outcome framing:
- stronger search visibility
- higher-intent acquisition through SEO
- better conversion support through cart recovery and loyalty mechanics
- stronger repeat engagement through system-led incentives

Notes:
- okay to mention abandoned cart recovery directly
- okay to mention loyalty logic directly
- okay to mention Meta / Instagram automations directly
- prefer saying "90%+ SEO health score" instead of a precise 91%

## Project 2: Kumbusha

Public name:
- Kumbusha

URL:
- https://kumbusha.vercel.app/

Type:
- in-progress culture / reminder workflow product

Status:
- in-progress venture
- some cake vendors have already been onboarded
- development is still ongoing
- company outreach and validation come after development milestones

Problem:
- managers often miss meaningful employee milestones because daily work takes priority
- employees can feel unseen or unappreciated when birthdays, anniversaries, promotion dates, and joining milestones go unnoticed

Core idea:
- a culture OS / milestone reminder system for teams

Workflow:
- managers upload employee data including important milestone dates
- system tracks upcoming events
- WhatsApp automations notify managers a set number of days before the milestone
- manager can take quick actions such as organizing a gift or scheduling a follow-up reminder
- when gift flow is selected, the system can route the request toward onboarded vendors with location/context details

Safe public framing:
- in-progress venture with real workflow logic in development
- early vendor onboarding has started
- product validation and outreach are part of the next phase

## Public Copy Guidance

When writing for the site:
- sound precise, confident, and operational
- avoid agency-style fluff
- emphasize systems, workflows, architecture, and measurable outcomes
- keep client-sensitive implementation details at a safe level
- case studies should explain why the system mattered, not just what screens were built
